# Novel Maker - AI 공동 집필 시스템

> AI 에이전트를 활용해 트렌디하면서도 완성도 높은 웹소설을 집필하기 위한 가이드 및 워크스페이스

## 핵심 전략: AI를 "공동 집필 파트너"로 활용하는 4단계

### 1단계 - 스토리 엔진 설정: "고정된 결말"과 "변동적 과정" 정의

이야기가 산으로 가지 않으려면 AI에게 처음부터 **최종 목적지(결말)**를 명확히 인지시켜야 합니다.

- **최종 목표(Goal) 명시**: "이 소설의 결말은 주인공이 [OO]를 달성하고 [OO]한 성장을 이루며 끝난다. 모든 에피소드는 이 결말을 향한 복선이어야 한다."
- **단계별 마일스톤**: 전체 구성을 기-승-전-결 4단계 또는 웹소설식 5단계(도입-전개-위기-절정-결말)로 나누고, 각 단계에서 반드시 일어나야 할 핵심 사건을 미리 '확정'해 둠

### 2단계 - 피드백 루프 시스템

독자 피드백과 최신 트렌드를 반영하는 **가변적 업데이트** 체계:

- **트렌드 주입 프롬프트**: "현재 2026년 웹소설 시장에서 유행하는 [하이퍼-전문직/시스템/주체적 로판] 트렌드를 이 로그라인에 결합해줘."
- **피드백 반영 매뉴얼**: 새로운 챕터를 쓰기 전, 이전 화 독자 반응을 AI에게 전달하고 결말과 모순되지 않는 범위 내에서 조정 요청

### 3단계 - 문체 가이드라인

AI 특유의 기계적 말투를 제거하는 방법:

- **대화 중심 구성**: 내레이션보다 캐릭터 간 티키타카(대화)를 7:3 비율로
- **인간적 불규칙성 추가**: 구어체, 적절한 신조어, 짧은 호흡의 문장 혼용
- **Show, don't tell**: 인물의 감정을 행동과 묘사로 전달
- **에디터 페르소나**: "10년 차 웹소설 전문 편집자" 역할 부여

### 4단계 - 마스터 프롬프트 구조

```
[역할 부여] 당신은 최신 웹소설 트렌드에 정통한 스타 작가이자 편집자입니다.

[스토리 정보]
- 로그라인: [작성하신 로그라인 삽입]
- 최종 결말: [반드시 도달해야 할 고정 결말]
- 주요 타겟: [예: 2030 직장인 남성 / 로판 마니아 여성 등]

[집필 지침]
- 구성: 매 회차 끝에 클리프행어(Cliffhanger) 절벽 엔딩을 만들어줘.
- 문체: 설명조를 피하고, 인물의 행동과 대화를 통해 상황을 보여줘.
- 업데이트: 독자 피드백을 전달하면, 전체 줄거리 개연성을 유지하며 실시간 조정해줘.
- 제약: 설정 오류(설붕)가 발생할 것 같으면 미리 경고해줘.
```

## 새 프로젝트 시작하기

1. `projects/` 아래에 새 폴더 생성 (예: `projects/나의_소설_제목/`)
2. `_template/` 의 파일들을 복사하여 프로젝트 폴더에 붙여넣기
3. `story_engine.md` 부터 채워나가기
4. 회차 집필 시 `chapter.md` 템플릿 사용
5. 독자 반응 수집 후 `feedback_loop.md` 에 기록하며 루프 반복

| 공통 | 클리프행어, 중독성 루프, 정서적 보상 |
