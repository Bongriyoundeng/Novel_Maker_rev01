# Novel Maker — AI 에이전틱 소설 집필 시스템

> 사령관(사용자)은 직접 쓰지 않습니다. **명령·승인·검수**만 합니다.
> 4개의 전문화 에이전트가 협력하여 고몰입도 웹소설을 자율 집필합니다.

## 퀵스타트: 새 프로젝트 시작하기

### Step 1 — 사령관이 직접 결정할 것 (시작 전 필수)
```
□ 고정 결말 — 구체적 장면 이미지 수준으로 작성
□ 핵심 테마 — 이 소설이 말하고자 하는 것
□ 주인공의 궁극적 목표(북극성) — 1화부터 결말까지 일관되게 흐를 목표
□ 주인공의 욕망(표면적/진짜)과 두려움
□ 주인공의 변화 저항 요소 — 어떤 습관/편견/두려움이 변화를 막는가?
□ 절대 발생 불가 사건 목록
```

### Step 2 — 프로젝트 폴더 생성
```
projects/ 아래에 새 폴더 생성
예: projects/나의_소설_제목/

_template/ 의 파일들을 복사하여 프로젝트 폴더에 붙여넣기

[폴더 구조]
나의_소설_제목/
├── story_bible.md
├── characters/
├── beats/
├── micro_summaries/
├── feedback/
└── 자료조사/          ← 반드시 생성. 집필 정확도의 핵심.
    ├── README.md
    ├── 인물_직업/
    ├── 장소_공간/
    ├── 시대_문화/
    └── 전문분야/
```

### Step 3 — 플롯 건축가에게 시놉시스 설계 요청
```
agents/01_plot_architect.md 의 프롬프트 사용
→ 사령관 승인 후 story_bible.md 초안 확정
```

### Step 4 — 세계관 제작자에게 설정 완성 요청
```
agents/02_world_builder.md 의 프롬프트 사용
→ 사령관 승인 후 story_bible.md 업데이트
```

### Step 2.5 — 자료조사 (필요 시, 집필 전 언제든)
```
자료조사/ 폴더에 조사 파일 작성
→ _template/자료조사/ 의 템플릿 활용
→ 세계관 제작자에게 제공하여 스토리 바이블 정확도 향상
→ 비트 시트에 해당 화 참조 파일 경로 기록
→ 산문 스타일리스트 호출 시 관련 자료 붙여넣기
```

### Step 5 — 화별 반복 사이클 시작
```
① 플롯 건축가: 비트 시트 작성 (beat_sheet.md)
     [이번 화 참조 자료 링크 포함]
          ↓ [사령관 승인]
② 산문 스타일리스트: 4단계 레이어 집필 (scene_layer.md)
     [자료조사/ 관련 파일 내용 제공]
          ↓
③ 서사 감사관: 감사 보고서
          ↓ [사령관 최종 확정]
④ 세계관 제작자: story_bible.md 업데이트
          ↓
⑤ 마이크로 요약 작성 (micro_summary.md)
          ↓ 다음 화 루프
```

## 핵심 원칙 6가지

### 1. 이야기의 본질 — 목표를 가진 인물의 기록
> "이야기란 어떤 목표를 가진 캐릭터가 있고, 그 목표를 이루는 것이 힘들지만 불가능하지는 않은 과정의 기록."

주인공의 '궁극적 목표(북극성)'를 먼저 확정하고, 모든 사건이 이 목표와 연결되는지 항상 확인합니다.

### 2. 인물 중심 서사의 법칙
소설의 동력은 사건이 아니라 **인물의 내면**에서 나옵니다.
인물은 사건에 끌려다니는 존재가 아니라, 갈등의 원인을 제공하고 난관을 극복하는 **주체**입니다.
인물은 변화에 저항(항상성)하며, 그 저항을 뚫고 나가는 사투가 소설의 재미입니다.

### 3. 12단계 서사 구조의 법칙
일상 세계 → 소명 → 조건 부재 → 열쇠 → 통과 → 시험 → 그림자 → 시련 → 보상 → 정면승부 → 각성 → 귀환.
5단계 마일스톤(도입-전개-위기-절정-결말)을 12단계로 세분화하여 독자의 몰입을 설계합니다.
세부 내용: `가이드라인/03_서사공학_기법.md`

### 4. 고정 결말의 법칙
결말을 먼저 확정하고, 과정을 에이전트에게 맡깁니다.
결말이 흔들리면 모든 서사가 무너집니다.

### 5. But/Therefore 인과의 법칙
모든 사건은 '하지만' 또는 '그러므로'로 연결되어야 합니다.
'그리고 나서' 나열은 지루한 소설의 신호입니다.

### 6. Show, Don't Tell의 법칙
"그는 화가 났다"가 아니라 "관자놀이 핏대가 실룩거렸다"로 씁니다.
감정은 설명하지 않고 신체 반응과 행동으로 보여줍니다.
3가지 묘사 유형(직접 설명 / 스스로 드러내기 / 간접 묘사)을 상황에 따라 선택합니다.
