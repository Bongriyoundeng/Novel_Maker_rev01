# Novel Maker — AI 에이전틱 소설 집필 시스템

> 사령관(사용자)은 직접 쓰지 않습니다. **명령·승인·검수**만 합니다.

## 퀵스타트: 새 프로젝트 시작하기

### Step 1 — 사령관이 직접 결정할 것 (시작 전 필수)
```
□ 고정 결말 — 구체적 장면 이미지 수준으로 작성
□ 핵심 테마 — 이 소설이 말하고자 하는 것
□ 주인공의 욕망과 두려움
□ 절대 발생 불가 사건 목록
```

### Step 2 — 프로젝트 폴더 생성
```
projects/ 아래에 새 폴더 생성
예: projects/나의_소설_제목/

_template/ 의 파일들을 복사하여 프로젝트 폴더에 붙여넣기
```

### Step 3 — 플롯 건축가에게 시놉시스 설계 요청
```
agents/01_plot_architect.md 의 프롬프트 사용
→ 사령관 승인 후 story_bible.md 초안 확정
```

### Step 4 — 세계관 제작자에게 설정 완성 요청
```
agents/02_world_builder.md 의 프롬프트 사용
→ 사령관 승인 후 story_bible.md 업데이트
```

### Step 5 — 화별 반복 사이클 시작
```
① 플롯 건축가: 비트 시트 (beat_sheet.md)
          ↓ [사령관 승인]
② 산문 스타일리스트: 4단계 레이어 집필 (scene_layer.md)
          ↓
③ 서사 감사관: 감사 보고서
          ↓ [사령관 최종 확정]
④ 세계관 제작자: story_bible.md 업데이트
          ↓
⑤ 마이크로 요약 작성 (micro_summary.md)
          ↓ 다음 화 루프
```

## 핵심 원칙 3가지

### 1. 고정 결말의 법칙
결말을 먼저 확정하고, 과정을 에이전트에게 맡깁니다.
결말이 흔들리면 모든 서사가 무너집니다.

### 2. But/Therefore 인과의 법칙
모든 사건은 '하지만' 또는 '그러므로'로 연결되어야 합니다.
'그리고 나서' 나열은 지루한 소설의 신호입니다.

### 3. Show, Don't Tell의 법칙
"그는 화가 났다"가 아니라 "관자놀이 핏대가 실룩거렸다"로 씁니다.
감정은 설명하지 않고 신체 반응과 행동으로 보여줍니다.
