# 가이드라인 3: 서사 공학 기법

> AI에게 "재미있게 써줘"는 효과가 없습니다.
> 재미를 만드는 **구체적인 서사 장치와 기법**을 에이전트의 워크플로우에 내재화해야 합니다.

---

## 기초 원칙: 이야기의 본질

> **"이야기란 어떤 목표를 가진 캐릭터가 있고, 그 목표를 이루는 것이 힘들지만 불가능하지는 않은 과정의 기록이다."**

이야기는 가래떡과 같습니다. 어떻게, 얼마나 정교한 마디로 자르느냐에 따라 서사의 맛이 달라집니다.
단순한 2단계(시작-끝)에서 시작해 12단계로 세분화될수록 독자의 몰입도와 개연성은 기하급수적으로 상승합니다.

---

## 핵심 기법 0: 12단계 서사 구조

> **모든 장편 서사의 기반.** 플롯 건축가는 이 구조를 시놉시스 설계의 뼈대로 사용합니다.

| 단계 | 명칭 | 핵심 기능 |
|------|------|---------|
| 1 | **일상 세계** | 주인공의 평범한 환경 — 이후 상실의 고통을 극대화하기 위해 독자에게 충분히 각인 |
| 2 | **모험의 소명** | 기존 평화를 깨는 결정적 사건 — 피할 수 없는 결핍 또는 강력한 동기 부여 |
| 3 | **조건의 부재** | 소명을 즉각 수행하기 힘든 심리적 저항·무력 상태 — 취약성 노출, 독자의 연민 유도 |
| 4 | **관문에 열쇠** | 첫 시도 또는 조력자 등장 — 해결의 실마리가 가시화되는 지점 |
| 5 | **첫 관문 통과** | 새 변화에 적응 시작 — 일시적 안도감, 독자가 "해결됐다"고 착각하게 |
| 6 | **시험** | 소소한 갈등과 일시적 안녕 — 본격 갈등 전 관계 구축, 폭풍 전야 |
| 7 | **그림자와의 조우** | 잠재된 핵심 공포·갈등이 수면 위로 — "해피엔딩이 가능할까?" 의구심 심기 |
| 8 | **시련** | 감정적 고통 최고조 — 주인공 신념이 무너지는 가장 어두운 밤 |
| 9 | **보상** | 시련 끝 새로운 시각·깨달음 — 외부 보상 아닌 **인식의 전환** |
| 10 | **정면승부** | 문제의 핵심과 대면 — 수동적 관찰자 → 능동적 해결사로 탈바꿈 |
| 11 | **각성** | 내면 성장 완성 — 작품의 주제 의식이 주인공 변화를 통해 투영 |
| 12 | **일상 귀환** | 변화 수용 후 새로운 일상 — 시작점보다 한 차원 높은 '뉴 노멀' |

### 12단계 자가 점검 체크리스트

```
□ 주인공의 최종 목표(북극성)가 1~12단계 전체에 일관되게 흐르는가?
□ 각 단계의 사건이 목표 달성을 방해하거나 돕는 직접적 장치인가?
□ 목표와 무관한 불필요한 조연이 서사 흐름을 방해하고 있지 않은가?
□ 결말에서 주인공의 내면 성장이 목표 성취와 맞닿아 있는가?
□ 각 단계가 'But/Therefore' 인과 관계로 연결되는가?
```

### 5단계 마일스톤 ↔ 12단계 서사 매핑

```
도입 (1단계) → 1단계(일상 세계) + 2단계(소명) + 3단계(조건 부재)
전개 (2단계) → 4단계(열쇠) + 5단계(통과) + 6단계(시험)
위기 (3단계) → 7단계(그림자) + 8단계(시련)
절정 (4단계) → 9단계(보상) + 10단계(정면승부)
결말 (5단계) → 11단계(각성) + 12단계(귀환)
```

## 핵심 기법 1: But/Therefore 프레임워크

### 원칙
모든 사건은 앞선 사건의 **결과**이거나 새로운 **장애물**이어야 합니다.

| ❌ And then (나열) | ✅ But/Therefore (인과) |
|-----------------|----------------------|
| 주인공이 던전에 들어갔다. 그리고 나서 몬스터를 만났다. | 주인공이 보물을 찾으러 던전에 들어갔다. **하지만** 예상보다 강한 몬스터가 막아섰다. **그러므로** 지하 수로로 우회했다. |
| 그는 밥을 먹었다. 그리고 나서 잠을 잤다. | 그는 독이 든 밥을 먹었다. **그러므로** 의식을 잃기 전에 해독제를 찾아야 했다. **하지만** 해독제는 적의 손에 있었다. |

### 적용 방법
```
비트 시트 작성 시 플롯 건축가에게 지시:
"모든 사건 연결에 '하지만/그러므로'를 사용해줘.
'그리고 나서'로 연결된 사건이 있으면 다시 써줘."
```

## 핵심 기법 2: Show, Don't Tell — 3가지 묘사 유형

### 원칙
독자가 인물의 감정을 **직접 느끼게** 해야 합니다.

### 묘사법 3가지 유형

| 유형 | 설명 | 사용 시점 |
|------|------|---------|
| **직접 설명 (Telling)** | 정보 전달 위주의 압축 서술: "그는 용감하다" | 중요도 낮은 정보를 빠르게 처리할 때 |
| **스스로 드러내기** | 인물의 대사·행동·선택을 통해 성격 암시 | 인물의 본질이 드러나는 결정적 장면 |
| **간접적 묘사** | 주변 인물의 반응·관찰을 통한 형상화 | 주인공 이외 시점에서 묘사할 때 |

### Tell vs Show 비교표

| ❌ Tell (설명) | ✅ Show (묘사) |
|-------------|-------------|
| "그는 화가 났다" | "관자놀이 핏대가 실룩거렸다. 손에 쥔 펜이 부러질 듯 소리를 냈다." |
| "그녀는 두려웠다" | "손끝이 발끝까지 식어들었다. 숨을 들이쉴 때마다 폐가 좁아지는 것 같았다." |
| "분위기가 무거웠다" | "아무도 숟가락 소리조차 내지 않았다. 국물이 식어가는 소리만 들렸다." |
| "그는 날씬하다" | "그가 몸을 돌리자 드레스 위로 앙상한 뼈마디가 도드라졌다." |
| "그는 용감하다" | "손떨림에도 불구하고 마스크를 조여 매고 연기 자욱한 구덩이 속으로 몸을 던졌다." |

> **효과적인 묘사 전략:** 모든 문장을 묘사로 채우면 서사가 늘어집니다. 중요도가 낮은 정보는 '설명'으로 빠르게 처리하고, 인물의 본질이 드러나는 결정적 장면에 '보여주기' 화력을 집중하십시오.

### 감각 동원 원칙
```
장면을 묘사할 때 5가지 감각을 고려합니다:
- 시각: 무엇이 보이는가?
- 청각: 무슨 소리가 들리는가?
- 촉각: 무엇이 느껴지는가?
- 후각: 무슨 냄새가 나는가?
- 미각: (필요한 경우)

모든 감각을 다 쓸 필요는 없습니다. 장면에서 가장 강렬한 1~2가지를 깊게 묘사합니다.
```

## 핵심 기법 3: 계층적 생성 프로세스 (4단계 레이어)

한 번에 모든 것을 생성하면 AI의 주의가 분산되어 각 요소의 밀도가 낮아집니다.
4단계로 나눠 생성하면 각 레이어의 완성도가 높아집니다.

```
1단계: 감각적 배경
  → 장면의 물리 환경만 (조명, 소리, 냄새, 분위기)
  → 아직 캐릭터 없음

2단계: 내면 목소리
  → POV 캐릭터의 심리, 욕망, 두려움 추가
  → 직접 서술 금지, 신체 반응 + 내적 독백으로

3단계: 대화 & 행동
  → 갈등 유발 또는 정보 전달 대화 배치
  → "그가 말했다" 대신 제스처/행동으로 대화 태그 대체

4단계: 최종 합성
  → 1~3단계를 리듬 있는 산문으로 통합
  → 문장 호흡 조절, 베이지색 표현 제거
```

자세한 사용법: `_template/scene_layer.md`

## 핵심 기법 4: 클리프행어 설계

### 클리프행어 유형

| 유형 | 설명 | 예시 |
|------|------|------|
| 정보 공개형 | 예상치 못한 사실 공개 | "그제서야 그녀는 깨달았다. 등 뒤의 그 사람이..." |
| 위기 고조형 | 더 큰 위험이 등장 | 해결했다고 생각한 순간 더 강한 적 등장 |
| 선택 압박형 | 돌이킬 수 없는 선택 앞에 세움 | "10초 안에 결정하지 않으면 모두 죽는다" |
| 관계 반전형 | 믿었던 인물의 배신 | "그가 내민 것은 총이었다" |

### 클리프행어 강도 테스트
```
"이 화를 읽은 독자가 다음 화를 보지 않을 수 있는가?"
→ "예" = 클리프행어 실패, 강화 필요
→ "아니오" = 클리프행어 성공
```

## 핵심 기법 5: 인물 형상화 — 6가지 특징 설계

> **소설의 동력은 사건이 아니라, 그 사건을 마주하는 '인물'의 내면에서 나옵니다.**
> 인물은 서사의 수동적 피해자가 아닌, 갈등의 원인을 제공하고 난관을 극복하는 **주체**여야 합니다.

| 특징 | 설명 | 예시 |
|------|------|------|
| **고정관념 활용** | 대중적 인식을 베이스로 삼되 비틀기 | '주토피아' 닉 — 여우는 꾀가 많다는 인상 활용 |
| **신체적 특징** | 시각적 전형성 — 독자가 기억할 포인트 | 해리포터 번개 흉터, 크루엘라 반반 머리색 |
| **말투와 행동** | 고유한 습관 설정 | 셜록 홈즈의 날카로운 추리력과 언변 |
| **상황 설정** | 인물이 처한 특수한 환경 | 몸이 작아진 탐정 코난, 구박받는 신데렐라 |
| **특별한 능력** | 비일상적인 힘 부여 | 거미에게 물린 스파이더맨, 막대한 재력의 아이언맨 |
| **태생적 특징** | 이름·혈통에 운명 심기 | '전지적 작가 시점' 김독자 — 이름 자체가 '독자' |

### 인물 형상화 주의사항
```
⚠️ 모든 설정을 채우려다 에너지를 낭비하지 마십시오.
오직 '스토리의 중심축(서사)'과 직접 연결되어
사건을 유발하거나 해결하는 데 기여하는 설정만을 최우선으로 배치해야 합니다.
```

## 핵심 기법 6: 캐릭터 욕망·두려움 & 갈등의 이중 구조

재미있는 소설의 본질은 **캐릭터의 욕망이 세계의 제약과 부딪히는 순간**에 있습니다.

```
주인공 설계 공식:
  표면적 욕망 + 진짜 욕망 + 가장 큰 두려움

예시:
  표면적 욕망: "강해지고 싶다"
  진짜 욕망:  "사랑하는 사람을 잃고 싶지 않다"
  두려움:     "결국 혼자 남겨질 것이다"
```

이 세 가지가 충돌할 때 독자는 캐릭터와 깊이 공감합니다.

### 갈등의 이중 구조

```
[내적 갈등]
→ 인물의 심리적 고뇌, 트라우마, 정신적 마찰

[외적 갈등]
→ 타인, 사회 체제, 적대자 등 외부 존재와의 마찰

[핵심 원리]
이 두 가지는 복합적으로 작용해야 합니다.
외적 사건은 내적 갈등을 건드리고,
변화된 내면은 다시 외적 선택에 영향을 줍니다.
```

### 사건 구성 5가지 체크리스트

갈등 상황에서 인물을 자연스럽게 움직이게 하려면:

```
□ 인물이 현재 할 수 있는 것과 할 수 없는 것은 무엇인가?
□ 상황에 임하는 인물의 생각과 태도는 어떠한가?
□ 인물이 현재 가장 간절히 바라는 것은 무엇인가?
  (예: 초능력이 생겼어도 가장 원하는 것이 '내 집 마련'일 수 있음)
□ 인물이 달성하고자 하는 구체적인 목표는 무엇인가?
□ 인물을 나아가게 하는 원동력(신념, 사랑, 결핍 등)은 무엇인가?
```

## 핵심 기법 7: 인물의 깊이 — 과거, 양면성, 히어로-빌런

> 진짜 살아있는 것처럼 느껴지는 인물을 만들려면 그들의 **과거를 정교하게 설계**해야 합니다.

### 과거 설정의 목적
```
과거 = 인물의 가치관 + 트라우마 + 결핍의 근거

"저 인물이라면 과거의 상처 때문에 충분히 그럴 수 있다"는
개연성에서 설득력 있는 반전이 나옵니다.
```

### 양면성과 입체적 인물
```
외적인 면모와 숨겨진 내적 과거가 충돌할 때 인물은 입체적으로 변합니다.

예: '오페라의 유령' 에릭
  외적: 잔인한 범죄자
  내적: 흉측한 외모로 어머니에게조차 버림받은 상처
  → 이 충돌이 비틀린 욕망에 깊이를 더함
```

### 히어로와 빌런 — 종이 한 장 차이

```
히어로와 빌런은 종종 동일한 트라우마와 상처를 공유합니다.

예시:
  토르와 로키 — 같은 환경에서 자라며 결핍을 느낌
  닥터 스트레인지의 히어로/빌런 — 같은 상처를 입음

[차이점]
위기 순간의 '선택'이 운명을 가릅니다.
  빌런: 상처를 이유로 악행을 정당화
  히어로: 고통 속에서도 끝내 선한 선택을 내림으로써 상처 극복

[매력적인 빌런 설계 원칙]
나름의 확고한 논리와 정당성을 갖추게 합니다.
독자에게 묘한 설득력과 공포를 동시에 주는 것이 목표입니다.
```

## 핵심 기법 8: 조연 & 악역(적대자) 운용 전략

### 비중 조절의 두 가지 레버

| 레버 | 설명 | 예시 |
|------|------|------|
| **스포트라이트** | 문장 내 언급 빈도 + 대사의 양 | 해리포터는 스포트라이트 최대 |
| **돋보기** | 가족관계·과거사 등 세부 정보 공개 정도 | 헤르미온느의 부모님 정보는 최소화 |

### 조력자 설계 원칙
```
조력자는 주인공에게 교훈을 주거나 동기부여를 제공합니다.

[핵심 기법]
주인공과 인생의 연대기(타임라인)가 다른 조력자를 활용하십시오.
'스파이더맨(피터 파커)'과 '아이언맨(토니 스타크)'처럼
서로 다른 궤적을 그리던 인물들이 만나고 헤어지는 타이밍을 설계하여
서사의 리듬을 조절합니다.
```

### 서브 스토리의 기능
```
조연의 이야기는 반드시:
  → 메인 스토리의 의미를 강화하거나
  → 주인공을 행동하게 만드는 촉매제로 기능해야 합니다.
```

## 핵심 기법 9: 인물의 변화와 항상성 (변화 저항의 원칙)

> **소설의 궁극적인 주제는 인물의 '변화'를 통해 구현됩니다.**

### 변화의 원리
```
인물은 이전까지 무시하던 중요한 가치를 깨달을 때 변화합니다.

예: 급하게 달리다 넘어진 사람이
  단순히 아파하는 것이 아니라,
  "아무리 급해도 신발 끈은 묶고 가야 한다"는 의미를 얻는 과정
  → 이것이 소설의 핵심입니다.
```

### 항상성 — 변화에 대한 저항

```
인간은 본능적으로 현상 유지를 원하며 변화에 저항합니다.
소설 속 인물 역시 습관·편견·두려움 때문에 변화를 거부하며 시행착오를 겪습니다.

[창작 원칙]
인물을 쉽게 변화시키지 마십시오.
저항의 크기가 클수록,
그것을 뚫고 나가는 사투가 치열할수록,
독자는 인물의 변화에서 강력한 서사적 카타르시스를 느낍니다.

그 저항이 곧 소설의 재미입니다.
```

## 핵심 기법 10: 사이다/고구마 구조 (한국 웹소설 리듬)

한국 웹소설 독자는 **긴장(고구마)과 해소(사이다)의 교차 리듬**을 기대합니다.
이 리듬이 없으면 독자는 "재미없다"고 느끼고 이탈합니다.

### 사이다/고구마 설계 공식

```
[고구마 설치]
→ 독자가 "왜 저렇게 행동하지?", "빨리 해결해!" 하고 답답함을 느끼게 만들기
→ 주인공이 억울하거나, 무시당하거나, 능력을 발휘 못하는 상황

[사이다 폭발]
→ 주인공이 상대를 정확히 압도하거나, 의외의 방식으로 문제를 해결하는 순간
→ 과하지 않게, 구체적으로. 독자가 "그래! 이거야!" 하고 느끼는 순간

[미결 훅 (다음 고구마 설치)]
→ 사이다 직후 새로운 의문·위협·선택지 등장
→ 독자가 "다음엔 어떻게 되지?"를 느끼게 만들기
```

### 능력 각성 장면의 욕망 연결 원칙

```
❌ 실패 패턴: "슛이 들어갔다. 기분이 좋았다."
→ 능력이 그냥 편리한 도구로 보인다.

✅ 성공 패턴: 능력이 주인공의 가장 큰 상처(두려움)와 연결될 때
→ 예: 학교 농구팀에서 떨어졌던 기억 + 체공 연장 첫 경험
→ "이게 있으면... 될 수도 있겠다." (말로 하지 말고 행동/신체 반응으로)

[원칙]
능력의 첫 경험은 반드시 캐릭터의 표면적 욕망이 아닌
'진짜 욕망 혹은 두려움'과 충돌해야 한다.
```

### 각 화 마지막 장면 설계 원칙

```
[깔끔하게 닫히면 독자가 떠난다]
→ "해결됐다. 다음날 또 만났다." → 이탈 유도

[미결로 열리면 독자가 남는다]
→ 의문 / 새 위기 / 예상치 못한 반응 중 하나를 심는다.
→ 마지막 대사나 행동 하나가 다음 화의 "왜?"를 만든다.

[클리프행어 테스트]
"이 화를 읽은 독자가 다음 화를 보지 않을 수 있는가?"
→ "예" = 클리프행어 실패
→ "아니오" = 클리프행어 성공
```

## 핵심 기법 11: 사령관을 위한 집필 명령 필터링 조건

> 단순 "작성 요청" 대신 아래 조건을 추가해 에이전트의 예측 가능성을 깨뜨리세요.

```
[필터링 조건 예시]

◆ 동사 중심 서술:
  "부사(매우, 갑자기 등) 사용을 금지하고,
   강한 동사와 구체적인 명사만 사용하여 장면을 묘사하라."

◆ 레이어드 집필:
  "먼저 장면의 '냄새·소리·온도'만 묘사한 뒤,
   그 위에 캐릭터의 심리적 압박감을 얹어라."

◆ 대화 서브텍스트:
  "캐릭터들이 속마음을 그대로 말하지 않게 하라.
   서로 말을 가로채거나, 엉뚱한 질문을 던지는 식으로 대화의 리듬을 불규칙하게 만들어라."

◆ 거친 질감 요구:
  "문장을 매끄럽게 다듬지 말고,
   인물의 당혹감과 공포가 문장의 호흡(단문과 파격적인 어휘)에서 드러나게 써라."

◆ 비릿한 승부욕 표현:
  "[캐릭터명]는 침착하지만 내면에 강한 승부욕이 있다는 설정을 반영하라.
   단순히 농구 공을 넣는 것이 아니라, 상대를 압도했을 때의
   짧고 날카로운 쾌감을 신체 반응으로 묘사하라."

◆ 인물 변화 항상성 명시:
  "인물이 쉽게 변화에 동의하지 않게 하라.
   습관과 편견, 두려움 때문에 한 번 이상 거부하고 시행착오를 겪게 하라."
```

## 호흡(Pacing) 조절 가이드

```
긴장 장면 → 문장 짧게 (1~5단어 문장 혼용)
감정 장면 → 문장 중간 길이, 감각 묘사 풍부하게
설명 구간 → 가급적 짧게, 대화에 녹여서
회상 장면 → 현재 시점보다 문체 느리게
액션 장면 → 극단적으로 짧은 문장, 동사 중심
```
