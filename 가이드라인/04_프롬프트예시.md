# 가이드라인 4: 실전 프롬프트 예시

> 실제 집필 과정에서 바로 복사해서 쓸 수 있는 프롬프트 모음입니다.

---

## 상황별 프롬프트 모음

### A. 프로젝트 첫 시작 (세계관 설정)

```
[역할]
당신은 최신 웹소설 트렌드에 정통한 스타 작가이자 10년 차 편집자입니다.

[소설 정보]
- 장르: [판타지 / 로판 / 현판 / 기타]
- 로그라인: [한두 문장 요약]
- 타겟: [예: 20-30대 여성 직장인, 로판 마니아]
- 고정 결말: [결말 내용]
- 플랫폼: [연재 플랫폼]

[세계관 설정 요청]
위 로그라인을 바탕으로 2026년 독자 취향에 맞는 세계관을 설계해줘.
- 마법/시스템 규칙 (명확한 한계 포함)
- 사회/계급 구조
- 주인공이 활동하는 공간
- 독자의 흥미를 끌 독창적 요소 1-2개

단, 결말에 명시한 내용과 모순이 생기면 먼저 알려줘.
```

---

### B. 1화 초안 작성

```
[역할] 위에서 설정한 세계관과 결말을 기억하세요.

[1화 목표]
- 주인공을 매력적으로 소개
- 세계관을 자연스럽게 설명 (설명하지 말고 보여줄 것)
- 독자가 다음 화를 클릭할 수밖에 없는 클리프행어

[문체]
- 대화/행동 중심 7:3
- 짧은 문장으로 빠른 호흡
- 1화 목표 분량: 약 [_]자

1화를 써줘.
```

---

### C. 중간 회차 (피드백 반영 포함)

```
[현재 상태]
- 현재 회차: _화
- 현재 스토리 단계: [도입/전개/위기/절정]
- 결말까지 남은 마일스톤: [다음 필수 이벤트]

[이전 화 마지막]
"[직전 화의 마지막 문장]"

[독자 피드백 반영]
- 좋았던 것: [독자 반응]
- 개선 요청: [독자 지적]
- 반영 방향: [어떻게 반영할지]

[이번 화 지시]
- 반드시 포함할 사건:
  1.
  2.
- 클리프행어:
- 목표 분량: _자

_화를 써줘. 세계관 규칙과 결말 방향을 유지해줘.
```

---

### D. 설붕 위험 경고 요청

```
지금까지 집필된 내용을 검토해서 다음 사항을 점검해줘:

1. 세계관 규칙 위반 가능성
2. 캐릭터 성격 일관성 오류
3. 복선이 회수되지 않은 항목
4. 결말 방향과 모순되는 전개

발견하면 어떤 문제인지, 어디서 발생했는지, 어떻게 수정하면 좋은지 제안해줘.
```

### E. 퇴고 및 문체 개선

```
[퇴고 요청]
당신은 10년 차 웹소설 전문 편집자입니다.

아래 원고를 퇴고해줘:
1. 기계적 표현 → 자연스러운 구어체
2. Tell → Show (감정을 행동으로)
3. 캐릭터 말투 일관성
4. 리듬감: 단조로운 문장 → 다양한 호흡
5. 불필요한 부사/중복 표현 제거

[원고]
(원고 붙여넣기)
```

---

### F. 클리프행어 개선

```
현재 작성한 _화 엔딩이 충분히 강렬하지 않아.
독자가 "다음 화를 반드시 봐야겠다"고 느낄 수 있는 클리프행어로 바꿔줘.

현재 엔딩:
"[현재 엔딩 내용]"

조건:
- 이미 설정된 _화의 다음 이벤트와 자연스럽게 연결될 것
- 과도한 충격보다 "궁금증 유발"에 집중
- 클리프행어 옵션 3가지를 제안해줘
```

```
[캐릭터 말투 설정]
- 지수: [말투 특징]
- 민준: [말투 특징]

[대화 원본]
(대화 붙여넣기)

캐릭터 말투 설정과 다른 부분을 찾아서 수정해줘.
```
