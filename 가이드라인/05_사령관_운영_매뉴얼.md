# 가이드라인 5: 사령관 운영 매뉴얼

> 사령관(사용자)이 에이전트 시스템을 운영하는 실전 가이드입니다.
> "어떻게 명령하고, 어떻게 검수하며, 어떻게 개입하는가"를 다룹니다.

---

## 사령관의 핵심 역할

```
집필하지 않는다.
에이전트의 결과물을 승인하거나 반려한다.
결정 경계 항목에서 최종 결정을 내린다.
이야기의 '영혼'(핵심 의도)을 지키는 것이 유일한 창작 역할이다.
```

---

## 프로젝트 시작 체크리스트

### 사령관이 직접 확정해야 할 것들

- [ ] **고정 결말**: 구체적 장면 이미지 수준으로 작성
- [ ] **핵심 테마**: 이 소설이 궁극적으로 말하고자 하는 것
- [ ] **주인공 욕망과 두려움**: 에이전트가 캐릭터를 이해하는 기반
- [ ] **절대 발생 불가 사건**: 사령관만 알고 있는 서사 경계

> 이것들은 AI에게 맡기면 안 됩니다. 사령관이 직접 결정해야 이야기에 '영혼'이 생깁니다.

---

## 에이전트 지휘 실전 가이드

### 좋은 명령 vs 나쁜 명령

| ❌ 나쁜 명령 | ✅ 좋은 명령 |
|-----------|-----------|
| "재미있게 써줘" | "3화에서 주인공이 처음으로 강적에게 패배하는 장면. 독자가 분노와 긴장을 동시에 느껴야 해." |
| "캐릭터를 매력적으로 만들어줘" | "민준의 냉소적 말투 뒤에 숨겨진 취약함을 독자만 알 수 있게 행동으로 보여줘." |
| "더 긴장감 있게 해줘" | "이 장면에서 주인공이 3초 안에 결정하지 않으면 조력자가 죽는 상황으로 만들어줘." |

### 에이전트 결과물 승인 기준

```
[승인] 조건:
- 고정 결말 방향과 일치한다
- '하지만/그러므로' 인과 관계가 있다
- 클리프행어가 있다

[반려] 조건:
- 결말과 모순된다
- 'And then' 나열만 있다
- 설붕이 포함되어 있다
- 사령관의 핵심 의도에서 벗어났다
```

## 결정 경계 운영

### 에이전트가 반드시 물어봐야 할 7가지

1. 주요 캐릭터의 사망 또는 영구 퇴장
2. 세계관 핵심 규칙 변경
3. 고정 결말 방향과 충돌하는 전개
4. 신규 주요 캐릭터 추가
5. 기존 캐릭터의 핵심 성격 반전
6. 10화 이상의 플롯 방향 전환
7. 독자 피드백이 결말 방향과 충돌

### 에이전트가 묻지 않고 결정해도 되는 것

- 장면 세부 묘사 방식
- 대화의 구체적 내용
- 부차적 캐릭터의 행동
- 문체와 호흡 선택
- 장면 순서 (비트 범위 내)

---

## 독자 피드백 처리 프로토콜

### 3단계 분류

**A. 즉시 반영**
```
- 문체가 딱딱하다는 지적
- 특정 캐릭터 대사가 어색하다는 피드백
- 오타 / 표현 오류
```

**B. 검토 후 반영**
```
- 특정 캐릭터 등장 빈도 조정 요청
- 전개 속도 조절 요청
→ 마일스톤과 충돌하지 않으면 반영
```

**C. 반영 불가 (결말 방향과 충돌)**
```
- 결말을 바꾸는 내용의 요청
- 핵심 마일스톤 이벤트 제거 요청
→ 반영하지 않되, 다른 방식으로 독자 보상 제공
```

### 피드백 반영 프롬프트
```
[피드백 반영 요청]

_화에 대한 독자 반응이 나왔어. 다음 화에 반영해줘.

[즉시 반영 항목]
-

[검토 후 반영 항목]
-

[유지해야 할 고정값]
- 최종 결말:
- 다음 마일스톤:

충돌 가능성이 있으면 먼저 나에게 알려줘.
```

## 사령관이 범하기 쉬운 실수

| 실수 | 결과 | 해결법 |
|------|------|--------|
| 에이전트에게 너무 많은 자유를 줌 | 결말에서 이탈, 설정 오류 | 결정 경계를 명확히 설정 |
| 독자 피드백을 무조건 반영 | 결말 개연성 붕괴 | A/B/C 3단계 분류 후 선택적 반영 |
| 비트 시트 없이 바로 집필 지시 | 구조 없는 에피소드 나열 | 항상 비트 승인 후 집필 지시 |
| story_bible.md 업데이트 미룸 | 설붕 누적 | 화 완료 즉시 업데이트 원칙 |
| 에이전트 결과물을 무조건 수용 | 품질 저하 | 서사 감사관 검수 후 최종 승인 |
